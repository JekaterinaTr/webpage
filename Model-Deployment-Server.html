Deployment Architecture

YOLO11 Model: Trained for pollen detection on dense microscopy images.
Server Stack: Ubuntu server + Python 3.11 + PyTorch + Flask API + Gunicorn + Nginx
Database: MongoDB for storing detection results, timestamps, and metadata
Automation: Cron jobs or Celery pipelines for scheduled inference

Workflow
  1. Upload images via API / batch directory.
  2. YOLO11 model inference on server GPU or CPU fallback.
  3. Store results (bounding boxes, class, confidence, timestamps) in MongoDB.
  4. Trigger downstream processing for analytics dashboards or CSV exports.
  5. Optional edge synchronization: model can run offline if needed.

Automation & Cloud Integration
  • Flask REST API exposes endpoints for image upload and inference requests.
  • Celery worker manages scheduled batch inference and retries for large datasets.
  • MongoDB stores results with indexing for fast retrieval and analytics.
  • Optional dashboard integration with Plotly / Dash or React for live visualization.
Business & Technical Impact
  • Enables continuous pollen monitoring with minimal human intervention.
  • Scalable to multiple cloud nodes or hybrid edge-cloud setups.

  • Foundation for integrating additional models or automated triggers for alerts.
